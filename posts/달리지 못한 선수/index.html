달리지 못한 선수(해쉬) 알고리즘

달리지 못한 선수

문제 설명

수많은 마라톤 선수들이 마라톤에 참여하였습니다. 단 한 명의 선수를 제외하고는 모든 선수가 마라톤을 완주하였습니다.

마라톤에 참여한 선수들의 이름이 담긴 배열 participant와 완주한 선수들의 이름이 담긴 배열 completion이 주어질 때, 완주하지 못한 선수의 이름을 return 하도록 solution 함수를 작성해주세요.

제한사항

  • 마라톤 경기에 참여한 선수의 수는 1명 이상 100,000명 이하입니다.
  • completion의 길이는 participant의 길이보다 1 작습니다.
  • 참가자의 이름은 1개 이상 20개 이하의 알파벳 소문자로 이루어져 있습니다.
  • 참가자 중에는 동명이인이 있을 수 있습니다.

입출력 예

participant completion retrun
[leo, kiki, eden] [eden, kiki] leo
[marina, josipa, nikola, vinko, filipa] [josipa, filipa, marina, nikola] vinko
[mislav, stanko, mislav, ana] [stanko, ana, mislav] mislav

입출력 예 설명

예제 #1 leo는 참여자 명단에는 있지만, 완주자 명단에는 없기 때문에 완주하지 못했습니다.

예제 #2 vinko는 참여자 명단에는 있지만, 완주자 명단에는 없기 때문에 완주하지 못했습니다.

예제 #3 mislav는 참여자 명단에는 두 명이 있지만, 완주자 명단에는 한 명밖에 없기 때문에 한명은 완주하지 못했습니다.

나의 풀이

def solution(participant, completion):
    answer = ''
    participant = sorted(participant)
    completion = sorted(completion)
    
    for number in range(len(participant)):
        
        if number == len(completion):
            answer = participant[number]
            break
        
        if participant[number] != completion[number]:
            answer = participant[number]
            break
        
    return answer

해당 코드는 최대한 {key : value} 형태를 만들어서 해쉬형태로 만들고 비교하는 방식으로 만들어 보았습니다. key는 선수이름이며, value는 참여자의 이름이 나온 횟수 입니다. 동명이인이 나오는 걸 체크하기 위한 방법으로 동명이인 존재 시 완주자의 해쉬값이 다른지를 확인하기 위해 저러한 형태로 만들었습니다. 생각해보니 앞 3자리만 slice해서 list로 만들었어도 됬었을지도?

능력자 분들의 갓 코드…(기록하고 싶은 것)

import collections


def solution(participant, completion):
    answer = collections.Counter(participant) - collections.Counter(completion)
    return list(answer.keys())[0]

collections 모듈이 있는지는 처음알았다. counter객체를 사용하면 list의 내용을 서로 빼는 것이 가능하다는 사실에 놀랐다…

def solution(participant, completion):
    answer = ''
    temp = 0
    dic = {}
    for part in participant:
        dic[hash(part)] = part
        temp += int(hash(part))
    for com in completion:
        temp -= hash(com)
    answer = dic[temp]

    return answer

정말 문제취지에 맞추어 해결해낸 답이 아닐까 싶다.
