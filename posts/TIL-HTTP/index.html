HTTP

HTTP란?

http : hypertext tranfer protocol의 약자로 HTML을 전송할때 공통된 규칙에 의해 서로 원할한 소통을 하기위해 만들어진 통신 규약이다. 예를 들면 말을 하게되면 서로가 알아들을 수 있어야 하지 않은가 말은 어떻게 하는가? 그냥 소리만 지른다고 알수는 없다. 한국어, 영어, 프랑스어 등 정해진 규칙, 문법에 따라 말해야할 것이다 HTML의 통신도 이와 비슷하다고 보면 된다.

즉 대상이 server가되고 server끼리는 공통의 언어로 말한다고 생각하면된다.

http에는 받는대상, 제목 , 주소, 내용, 보내는 대상들이 들어가있어 마치 편지와 비슷하다고 볼 수있다.

HTTP는 TCP/IP의 기반위에 만들어져 있다.

HTTP의 핵심요소

HTTP에는 2가지 큰 특징이 있다.

  1. HTTP는 기본적으로 요청/응답(request/response)구조로 되어있다.

    • 클라이언트가 rquest를 보내면 서버는 responser를 보내는 구조다
    • 모든 통신이 요청과 응답으로 이루어 진다.
  2. HTTP는 Stateless 이다. 말 그대로 상태를 저장하지 않는다는 뜻으로 독립적인 통신을 하므로 현재 이루어진 통신은 이전의 통신을 기억하지 못한다 즉 내가 내용을 보내서 인증 처리했더라도 이 통신이 끝나면 그것으로 끝이다 지속적으로 유지하려고 해도 다시 인증시켜 주어야한다. 대표적인 예로는 로그인이 있다 로그인은 로그인에 필요한 id / password를 서버에 보내고 인증을 받은 뒤 통신을 끝내면 끝이다 Stateless임으로 통신내용을 저장하지 않기 때문이다. 그래서 지속적인 로그인 정보를 요구 할것이다. 그래서 쿠키나 세션이라는 곳에 로그인 정보(토큰값)을 저장하여 요청에 대한 정보를 담고있다가 매 통신시 HTTP header나 body에 첨부하여 보내게된다. (이러한 정보는 성격상 header에 붙는게 맞다)

HTTP의 구조

HTTP는 request(요청), respose(응답) 2가지 경우가 있고 2가지 경우는 각각 3부분으로 구성되어 있다.

HTTP Request 메시지

  • status line
  • headers
  • body

Request의 큰 구조이다

  1. status line

    • HTTP Method 요청의 행동을 정의한다. (get-데이터 줘, post - 데이터 생성해)
    • Request target 보낼 주소로 uri를 말하며 큰 주소뒤에 붙는 상세주소라고 보면 쉽다. (ex : www.tset.com/login /login부분이라고 보면된다.)

    • http viersion http의 버전 주로 1.1대를 사용 중이고 간혹 2사용하긴한다, 현재 3버전까지 나옴

      GET /search HTTP/1.1

      위와 같은 형태가 된다.

  2. Headers

    • 메타정보, 요청에대한 부가정보로 되어있음, key:value의 형태로 되어있다.
    • Headers도 크게 3부분으로 나뉘어진다.(general headers, request headers, entity headers)

    • 자주 사용되는 header 정보들이 있다.

      • host : google.com / 김 아파트 라는 아파트로 가야함 근데 어디로 가는지 모름(동면읍, 몇동, 몇호) 그래서 status line의 target이 알려준다.
    • User-Agent : 클라이언트 정보 담김
    • Connection : 요청 끝난후 클라이언트와 서버 계속 연결 유지할지 여부
    • Content-Type : 해당 요청이 보내는 메세지 body의 타입. 예를 들어, JSON을 보내면 application/jso
    • Content-Length: 메세지 body의 길이
Accept: */*
Accept-Encoding: gzip, deflate
Connection: keep-alive
Content-Type: application/json
Content-Length: 257
Host: google.com
User-Agent: HTTPie/0.9.3

  1. body

    • 해당 reqeust의 실제 내용들이 담긴다.
    • Body가 없는 request도 많은데 GET 요청이 보통 정보를 요구하기 때문에 내용이 없는 경우가 많다.
{
    "imp_uid": "imp_1234567890",
    "merchant_uid": "order_id_8237352",
    "status": "paid"
}

  • Status line
  • Headers
  • Body Response의 큰 구조이다. Request와 비슷한 구조이다.

  • Status line

    • http viersion Request와는 다르게 앞에 온다.
    • Status code 응답상태 나타내는 숫자코드 ex) 200

    • Status text 응답 상태 간략히 설정 ex)Not Found

      HTTP/1.1 404 Not Found

  • Heder

    • Response의 headers와 동일하다. -다만 response에서만 사용되는 header 값들이 있다 예로는 User-Agent 대신 Server 헤더가 사용된다.

  • Body

    • Response의 body와 일반적으로 동일하다. 요청에대한 값은 보냄

    • Request와 마찬가지로 모든 body가 있지않고 데이터 전송 할 필요 없다면 비어있게된다.
