DataBase의 JOIN

관계형 데이터베이스는 데이터의 중복을 피하기위해 테이블을 여러개로 나누어 저장하고 있는데 이렇게 나누어져 있는 테이블들 중 2개이상의 테이블을 합쳐서 사용하기 위한 명령어이다.

보통 Primary Key나 Foreign Key로 테이블을 연결한다.

JOIN은 몇가지 유형으로 구성되어있다.

  • INNER JOIN(내부 조인)
  • LEFT OUTER JOIN (왼쪽 외부 조인)
  • RIGHT OUTER JOIN (오른쪽 외부 조인)
  • FULL OUTER JOIN ()
  • CROSS JOIN
  • SELF JOIN

등등이 있으나 자주 쓰이는 것 위주로 보도록 하겠다.

INNER JOIN(내부조인)

간단하게 수학의 교집합이라고 생각하면 쉽다. INNTER JOIN은 가장 많이 사용되는 JOIN구문 중 하나 이며 조건문에 따라 2개의 테이블(A,B) 컬럼을 합쳐 새로운 테이블을 만든다. 즉, A, B 테이블 둘다 가지고 있는 데이터만 검색되는 것이다.

# 1
SELECT [별칭1].[조회할 컬럼],
       [별칭2].[조회할 컬럼]
FROM [테이블명1] [별칭1]
INNER JOIN [테이블명2] [별칭2] ON [별칭1].[컬럼명] = [별칭2].[컬럼명];

# 2
SELECT [테이블명1].[조회할 컬럼],
       [테이블명2].[조회할 컬럼]
FROM [테이블명1], [테이블명2]
WHERE [테이블명1].[컬럼명] = [테이블명2].[컬럼명];

#EX
SELECT AA.NAME,
       BB.AGE,
FROM WHO AA
INNER JOIN MAN BB ON AA.NO_EXP = BB.NO_EMP;

위는 SQL문으로 INNER JOIN을 사용하는 법이다.

위의 예를 보면 #1은 명시적인 사용법이며, #2 처럼 암묵적인 사용법이 있다. 차이점은 INNER JOIN 과 ON을 사용하는가 WHERE절을 사용했는가 이다.

예를 보면 WHO테이블에서 NAME을, MAN테이블에서는 AGE를 조회하고 싶다. 그 중 NO_EXP라는 컬럼의 교집합을 통해서 보여준다.

LEFT OUTER(왼쪽 외부 조인)

왼쪽 외부 조인은 테이블 A의 모든 데이터와 B테이블과 매칭이 되는 레코드를 포함하는 JOIN이다.

# 1
SELECT [별칭1].[조회할 컬럼],
       [별칭2].[조회할 컬럼]
FROM [테이블명1] [별칭1]
LEFT OUTER JOIN [테이블명2] [별칭2] ON [별칭1].[컬럼명] = [별칭2].[컬럼명];

#EX1
SELECT AA.NAME,
       BB.AGE,
FROM WHO AA
LEFT OUTER JOIN MAN BB ON AA.NO_EXP = BB.NO_EMP;

#EX2
SELECT *
FROM table1
LEFT OUTER JOIN table2
ON table1.n = table2.n;

이번에는 LEFT OUTER JOIN으로 INNTER JOIN과 사용법에서 큰차이는 없다.

현재 DB모델을 해놓은것이 없어 블로그에 간단하게 예시가 잘 나와있어서 참고하였다. table1의 모든 데이터를 불러오며 중복되는 n컬럼을 출력해오겠단 뜻이다. table1과 table2의 n컬럼이 중 두 테이블에 동시에 존재하는 값은 2밖에 없으므로 위와 같은 결과가 나온것이다.

RIGHT OUTER JOIN(오른쪽 외부 조인)

RIGHT OUTER JOIN은 LEFT와 반대라고 생각하면된다. B테이블의 모든 데이터와 A테이블과 매칭되는 레코드를 포함한다.

# 1
SELECT [별칭1].[조회할 컬럼],
       [별칭2].[조회할 컬럼]
FROM [테이블명1] [별칭1]
RIGHT OUTER JOIN [테이블명2] [별칭2] ON [별칭1].[컬럼명] = [별칭2].[컬럼명];

#EX1
SELECT AA.NAME,
       BB.AGE,
FROM WHO AA
RIGHT OUTER JOIN MAN BB ON AA.NO_EXP = BB.NO_EMP;

#EX2
SELECT *
FROM table1
RIGHT OUTER JOIN table2
ON table1.n = table2.n;

LEFT JOIN처럼 사용해 주면된다.

예시도 정반대로 하면된다. table2의 모든 데이터를 불러오며 중복되는 n컬럼을 출력해오겠단 뜻이다. table1과 table2의 n컬럼이 중 두 테이블에 동시에 존재하는 값은 2밖에 없으므로 위와 같은 결과가 나온것이다.

FULL OUTER JOIN(완전 외부 조인)

FULL OUTER JOIN은 합집합이라고 생각하면된다. 모든 데이터가 다 나오게 된다.

# 1
SELECT [별칭1].[조회할 컬럼],
       [별칭2].[조회할 컬럼]
FROM [테이블명1] [별칭1]
FULL OUTER JOIN [테이블명2] [별칭2] ON [별칭1].[컬럼명] = [별칭2].[컬럼명];

# 2
SELECT *
FROM table1
  LEFT OUTER JOIN table2
    ON table1.n = table2.n
UNION
SELECT *
FROM table1
  RIGHT OUTER JOIN table2
    ON table1.n = table2.n;

# 3
SELECT *
FROM table1
  LEFT OUTER JOIN table2
    ON table1.n = table2.n
UNION ALL
SELECT *
FROM table1
  RIGHT OUTER JOIN table2
    ON table1.n = table2.n
WHERE table1.n IS null;

#EX1
SELECT *
FROM table1
FULL OUTER JOIN table2 ON table1.n = table2.n;

FULL OUTER JOIN은 몇 가지 방식으로 사용할 수 있다.

  1. 1 FULL OUTER JOIN을 사용한 방식

  2. 2 JOIN과 UINION을 사용한 방식

  3. 3 UNION ALL 과 exclusion join을 사용한 방식이다.

그리고 FULLOUTER를 사용하면 위의 예시 이미지처럼 전부 다 나오게되며 중복인 부분은 겹쳐서 나온다. 나머지 부분은 자동적으로 NULL이 된다.

CROSS JOIN(교차 조인)

CROSS JOIN은 가능한 모든 경우의 수를 전부 표현해 주는 방식이다. 즉 A테이블의 각행과 B테이블의 각 행을 다 조합한 결과이다.

# 1
SELECT [별칭1].[조회할 컬럼],
       [별칭2].[조회할 컬럼]
FROM [테이블명1] [별칭1]
CROSS JOIN [테이블명2] [별칭2]

# 2
SELECT [별칭1].[조회할 컬럼],
       [별칭2].[조회할 컬럼]
FROM [테이블명1] [별칭1], [테이블명2] [별칭2]

SELF JOIN(셀프 조인)

SELF JOIN은 자기 자신과 조인하는 조인으로 하나의 테이블을 여러번 복사해서 조인하다. 예를 들어 임직원중 같은 부서에서 일하는 직원을 알고 싶을때 사용하면 좋다

# 1
SELECT [별칭1].[조회할 컬럼],
       [별칭2].[조회할 컬럼]
FROM [테이블명1] [별칭1], [테이블명1] [별칭2]
WHERE [별칭1].[컬럼명] = [별칭2].[컬럼명]

# 2
SELECT [별칭1].[조회할 컬럼], [별칭2].[조회할 컬럼]
FROM [테이블명1] AS [별칭1] 
JOIN  [테이블명2] AS [별칭2] 
ON [별칭1].[컬럼명] = [별칭2].[컬럼명]

#EX
SELECT A.CustomerName AS CustomerName1, B.CustomerName AS CustomerName2, A.City
FROM Customers A, Customers B
WHERE A.CustomerID <> B.CustomerID
AND A.City = B.City
ORDER BY A.City;

예시는 W3에 있던 예시이며 WHERE절이후 조건을 주어 데이터를 가지고 온다. <>는 !=와 같은 뜻이다.
