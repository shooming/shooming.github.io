# mongoDB 설치
사용하기 위해 가장 먼저 해야할 일은 mongoDB를 설치하는 것 일것이다.
블로그등 올라와 있는 글들이 많지만 최신이 아닐 경우가 있기도 할 수 있으므로, 가장 좋은
방법은 공식 사이트 설치 방법을 따르는 방법이 가장 좋다.

## 공식 사이트를 통한 설치
1. [https://docs.mongodb.com/manual/administration/install-community/](https://docs.mongodb.com/manual/administration/install-community/)에 접속해서 server ccommunity 버전을 설치하는 법을 확인 할 수 있다. os에 맞게 설치 하는 법이 나와 있으며, linux의 ubuntu버전 기준으로 설치 가이드를 작성한다.

2. 패키지 관리 시스템에서 사용하는 공개 키를 가지고와야한다.
    ```bash
    wget -qO-https://www.mongodb.org/static/pgp/server-4.2.asc | sudo apt-key add-
    ```
    만약 gnumpg가 설치되어 있지않다면 오류가 발생할 수 있다. 그러므로 에러 메세지에 gnumpg 관련 오류 메시지가 나온다면 아래의 명령어를 입력해서 설치 하도록 한다.
    ```bash
    sudo apt-get install gnupg
    ```    
    ```bash
    wget -qO - https://www.mongodb.org/static/pgp/server-4.2.asc | sudo apt-key add -
    ```
3. MongoDB의 목록 파일을 작성한다.
    본인이 사용하는 ubuntu 버전에 맞게 `/etc/apt/sources.list.d/mongodb-org-4.2.list`에 리스트 파일을 만든다.
    ```bash
    echo "deb [ arch=amd64,arm64 ] https://repo.mongodb.org/apt/ubuntu bionic/mongodb-org/4.2 multiverse" | sudo tee /etc/apt/sources.list.d/mongodb-org-4.2.list
    ```

4. local package database를 다시 불러온다.
    ```bash
    sudo apt-get update
    ```

5. MongoDB packages를 설치한다.
    ```bash
    sudo apt-get install -y mongodb-org
    ```

## 공식 사이트를 통해 설치했으나 오류로 인해 사용불가시
나의 경우 공식 사이트 방식대로 설치를 했으나 의존성 패키지 관련 에러로 다시 설치도 삭제도 못하는 상태가 되었다가 --purge로 강제로 지워서 어떻게 사용할 수 있게 만들어놓았지만 다시 설치는 못해서 docker위에 올리는 방식을 선택했다.

```bash
sudo docker pull mongo
```
해당 명령어는 mogo의 offical image를 pull 받는 명령어 이다.
그 다음은 간단하다.

```bash
sudo docker run --name (원하는 이름) -d -p 27017:27017 mongo --auth
```
위 명령어를 통해 docker contaioner를 실행시키면된다.
