# mongoDB의 Database, Collection, Document 생성,제거

## Database의 생성 / 제거

### Database의 생성( use Database_name )
`use DATABASE_NAME`을 사용하면 database가 존재하지 않는다면 새로 생성하게 된다.
만약 존재한다면 해당 database내부로 진입하게 된다.

새로운 db 생성 혹은 해당 db 접속

```bash
> use test
switched to db test
```
현재 사용중인 데이터베이스 확인
```bash
> db
test
```

db 전체 조회 아래는 현재 내 mongoDB의 db조회 상황이다.
```bash
> show dbs
admin   0.000GB
config  0.000GB
local   0.000GB
prac    0.002GB

> show databases
admin   0.000GB
config  0.000GB
local   0.000GB
prac    0.002GB
```

현재 `test`db에 접속해 있지만 전체목록에서는 보이지 않는다. 그 이유는 Document를 하나라도 추가해야하기 때문이다.

```bash
> db.test.insert({"name" : "아뇽하세용", "age" : "999"})
WriteResult({ "nInserted" : 1 })

> show dbs
admin   0.000GB
config  0.000GB
local   0.000GB
prac    0.002GB
test    0.000GB
```
test가 새로 생겼다.

### Database의 제거( db.dropDatabase() )

database를 제거하려면 삭제하려는 db에 접속해주어야합니다. `use`로 switch 시켜주어야합니다.

삭제하려는 db에 들어왔다면 `db.dropDatabase()`를 사용합니다.

```bash
> use test

> db.dropDatabase();
{ "dropped" : "test", "ok" : 1 }
```

## Collection 생성 / 제거

### Collection 생성 ( db.createCollection() )

Collection을 생성할때는 db.createCollection(name,[option]) 명령어를 사용한다.

name에는 생성하려는 컬렉션 이름을 나타낸다.

option은 document 타입으로 구성된 해당 컬렉션의 설정값으로 선택적인 매개변수이므로 생략해도되며 필요에 따라 사용하면된다.

#### Option
|   Field   	|   Type  	|                                                                                                                     설명                                                                                                                    	|
|:---------:	|:-------:	|:-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------:	|
|   capped  	| Boolean 	| 이 값을 true 로 설정하면 capped collection 을 활성화 시킵니다. Capped collection 이란, 고정된 크기(fixed size) 를 가진 컬렉션으로서, size 가 초과되면 가장 오래된 데이터를 덮어씁니다. 이 값을 true로 설정하면 size 값을 꼭 설정해야합니다. 	|
| autoIndex 	| Boolean 	| 이 값을 true로 설정하면, _id 필드에 index를 자동으로 생성합니다. 기본값은 false 입니다.                                                                                                                                                     	|
|    size   	|  number 	| Capped collection 을 위해 해당 컬렉션의 최대 사이즈(maximum size)를 ~ bytes로 지정합니다.                                                                                                                                                   	|
|    max    	|  number 	| 해당 컬렉션에 추가 할 수 있는 최대 갯수를 설정합니다.                                                                                                                                                                                       	|

collection 만들고 조회해보기
```bash
> db.createCollection('호박고구마')
{ "ok" : 1 }

> show collections
호박고구마
```

option 넣을시
```bash
db.createCollection("호박고구마", {
... capped: true,
... autoIndex: true,
... size: 123456,
... max: 10000
... })
```
그리고 document를 추가하면서 자동으로 생성시킬 수도 있다.

```bash
> db.메타톤.insert({"name":"간장게장"})
WriteResult({ "nInserted" : 1 })

> show collections
메타톤
호박고구마
```

### Coleection 제거 (db.COLLECTION_NAME.drop())

제거할 때는 database때와 같이 `drop()`을 사용하면된다.

```bash
> show collections
메타톤
호박고구마
> 
> db.메타톤.drop()
true
```
## Document 생성 / 제거

### Document 생성 (db.COLLECTION_NAME.insert(내용) )

Document는 `insert()` 메소드를 사용한다.

```bash
> db.gugu.insert({'name' : '고등어'})
WriteResult({ "nInserted" : 1 })

> db.gugu.find({})
{ "_id" : ObjectId("5ed232c5dea93c027bea626f"), "name" : "고등어" }
```

### Document 제거 ( db.COLLECTION_NAME.remove(criteria, justOne) )

`remove(criteria, justOne)` 메소드를 사용하여 document 제거 한다.

|   Field  	|   Type   	|                                                                                   설명                                                                                   	|
|:--------:	|:--------:	|:------------------------------------------------------------------------------------------------------------------------------------------------------------------------:	|
| criteria 	| document 	| 삭제 할 데이터의 기준 값 (criteria) 입니다. 이 값이 { } 이면 컬렉션의 모든 데이터를 제거합니다.                                                                          	|
| justOne  	| boolean  	| 선택적(Optional) 매개변수이며 이 값이 true 면 1개 의 다큐먼트만 제거합니다. 이 매개변수가 생략되면 기본값은 false 로 서, criteria에 해당되는 모든 다큐먼트를 제거합니다. 	|

```bash
> db.gugu.find({"name" : "고등어"})
{ "_id" : ObjectId("5ed232c5dea93c027bea626f"), "name" : "고등어" }

> db.gugu.remove({"name" : "고등어"})
WriteResult({ "nRemoved" : 1 })

> db.gugu.find({"name" : "고등어"})
> 
> db.gugu.find({})
> 
```
