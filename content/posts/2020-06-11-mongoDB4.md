# mongoDB Document 내용 수정하기
Document은 `.update()`메소드를 사용하면 수정하여 줄 수 있다.

## Document 수정하기 ( db.COLLECTION_NAME.update() )

```bash
db.collection.update(
   <query>,
   <update>,
   {
     upsert: <boolean>,
     multi: <boolean>,
     writeConcern: <document>
   }
)
```
`update()`는 기본적으로 단 하나의 document를 수정하며, filed를 수정하거나 이미 존재하는 document를 대체할 수 있다.

|   Parameter  |   Type   |                                                                                설명                                                                               |
|:------------:|:--------:|-----------------------------------------------------------------------------------------------------------------------------------------------------------------|
| *query       | document | 업데이트 할 document의 criteria 를 정합니다. find() 메소드 에서 사용하는 query 와 같습니다.                                                                       |
| *update      | document | document에 적용할 변동사항입니다.                                                                                                                                 |
| upsert       | boolean  | Optional. (기본값: false) 이 값이 true 로 설정되면 query한 document가 없을 경우, 새로운 document를 추가합니다.                                                    |
| multi        | boolean  | Optional. (기본값: false)  이 값이 true 로 설정되면, 여러개의 document 를 수정합니다.                                                                             |
| writeConcern | document | Optional.  wtimeout 등 document 업데이트 할 때 필요한 설정값입니다. 기본 writeConcern을 사용하려면 이 파라미터를 생략하세요.|

### 특정 field 업데이트

```bash
# 회사의 지역을 "강남"으로 변경한다.
> db.company.update({ name : "test company", { $set: { location : "강남" } } })
WriteResult({ "nMatched" : 1, "nUpserted" : 0, "nModified" : 1 })
```
특정 field값을 수정할 땐 `$set` 연산자를 사용한다. 이 연자를 사용하여 새로운 field를 추가 할 수 있다.

### document를 replace하기

```bash
# new company의를 새로운 document로 교체한다.
> db.company.update({ name : "new company" }, { "name" : "new new company", "location" : "판교" } )
WriteResult({ "nMatched" : 1, "nUpserted" : 0, "nModified" : 1 })
```

### 특정 field 제거하기
```bash
# Founded의 field를 제거한다.
> db.company.update( {name : "gogo company"}, { $unset : {"Founded" : 1}} )
WriteResult({ "nMatched" : 1, "nUpserted" : 0, "nModified" : 1 })
```

### criteria(기준)에 해당되는 document가 존재하지 않으면 새로 추가하기
```bash
# upsert 옵션 설정하여 haha company가 존재하지 않는다면 새로 추가한다.
> db.company.update({ name : "haha company" }, { name : "haha company", "location" : "역삼"}, { upsert : true})
```

### 여러 document의 특정 field를 수정하기
```bash
# location이 강남인 document의 Founded를 2010으로 설정
> db.company.update(
    { location : { $eq = '강남'} },
    { $set : { Founded : 2010 }},
    { multi : true}
)
WriteResult({ "nMatched" : 2, "nUpserted" : 0, "nModified" : 0 })
```

### 배열 에 값 추가하기
```bash
# test company document의 employees 배열에 "김코코" 추가
> db.company.update(
... { name: "test company" },
... { $push: { employees: "김코코" } }
... )
WriteResult({ "nMatched" : 1, "nUpserted" : 0, "nModified" : 1 })
```

### 배열에 값 여러개 추가하기 + 오름차순으로 정렬하기
```bash
# Charlie document의 employees에 "사마의" 와 "미믹" 을 추가하고 알파벳순으로 정렬
> db.company.update(
... { name: "test company" },
... { $push: {
...     employees: {
...         $each: [ "사마의", "미믹" ],
...         $sort: 1
...     }
...   }
... }
... )
WriteResult({ "nMatched" : 1, "nUpserted" : 0, "nModified" : 1 })
```
$sort 값을 내림차순으로 정렬하려면 -1 로 하면 된다.
배열이 document의 배열이고 그 embedded document의 특정 field에 따라서 정렬을 할 때는 다음과 같이 설정하면 된다.
```bash
$sort: { KEY: 1 }
```

### 배열에 값 제거하기

```bash
# test company document에서 employees 값의 미믹 제거
> db.company.update(
... { name: "test company" },
... { $pull: { employees : "미믹" } }
... )
```

### 배열에서 값 여러개 제거하기
```bash
# test company document에서 employees 배열 중 "사마의" 와 "김코코" 제거
> db.company.update(
... { name: "test company" },
... { $pull: { employees: { $in: ["사마의", "김코코" ] } } }
... )
WriteResult({ "nMatched" : 1, "nUpserted" : 0, "nModified" : 1 })
```
