# 초기세팅

## 서버 처음 사용시 기본적인 세팅부분

1. .gitignore

git을 이용한 협업을 주로하기 때문에 git에 올리기 싫은 내용이거나 보안에 문제가되는 요소들은 사전에 필터링하여 git에 업로드 되지 않게 하기 위해 `.gitignore`이라는 파일을 작성하여 사용한다.

`.gitignore`는 https://www.gitignore.io/ 과 같이 자신이 사용하는 환경(linx, django, visualstudio code 등)을 적으면 자동으로 만들어 주는 사이트가 있다. 여기서 자신의 환경 적고 나온 내용을 복사해서 manage.py가 있는 폴더에 작성한다.

```bash
#.gitignor

### Django ###
*.log
*.pot
*.pyc
__pycache__/
local_settings.py
db.sqlite3
db.sqlite3-journal
media
my_settings.py  #<-- 이 부분 추가
```
위와 같이 기본적인 예외처리를 해주고 나서 마지막부분처럼 `.gitignore`의 django부분에 `my_settings.py`를 추가 시켜주어야한다. 왜냐하면 보안과 관련된 내용을 `my_settings.py`에 작성할 것이기 때문에 git에 올리지 안되기 때문이다.

## corsheaders
과거에는 django에서 페이지를 제공했었는데 그 당시 서버는 사용자의 웹브라우저를 신용하는 상태였다. 그러다 보니 공격자들이 게시물들을 활용하여 이미지와 같이 `scr='나쁜주소'` 웹서버가 주소를 참고 할 수 밖에 없게 하므로서 대상 게시물을 열어본 사용자를 통해 원치 않은 공격을 시작한다.

그걸 방지 위한 django의 장치가 csrf였던 것이었던 것이다. 그러나 시간이 흐르면서 동일 출처 정책 등 자바스크립트의 발전으로 필요가 없게 되었다. 그러다 보니 자신의 도메인이 아니면 다른 사이트의 정보를 받아오기 어려워져 이것을 해결하기위해 cors를 사용하게 된것이다. cors를 사용하여 허가된 출저 집단을 저장해서 http의 헤더에 담아서 사용한다. cross-domain과 같은 일들에 능동적으로 대응할 수 있게된다.

### cors 설치

```python
pip install django-cors-headers
```
pip를 이용해 conda 가상환경에 cors를 설치해 줍니다.

```python
# settings.py

INSTALLED_APPS = [
		'django.contrib.staticfiles',
		'corsheaders'
]

MIDDLEWARE = [
		'corsheaders.middleware.CorsMiddleware',
]
```
cors를 설치했으니 django에게 이 앱이 설치됬음을 이해 시켜주어야합니다.
INSTALLED_APPS에 등록하고 미들웨어가 입,출력제어를 해줄 수 있게 하기위해서
MIDDLWARE에 등록해준다.

```python
##CORS
CORS_ORIGIN_ALLOW_ALL=True
CORS_ALLOW_CREDENTIALS = True

CORS_ALLOW_METHODS = (
    'DELETE',
    'GET',
    'OPTIONS',
    'PATCH',
    'POST',
    'PUT',
)

CORS_ALLOW_HEADERS = (
    'accept',
    'accept-encoding',
    'authorization',
    'content-type',
    'dnt',
    'origin',
    'user-agent',
    'x-csrftoken',
    'x-requested-with',
		#만약 허용해야할 추가적인 헤더키가 있다면?(사용자정의 키) 여기에 추가하면 됩니다.
)
```
그리고 setting 마지막에는 cors의 옵션들을 작성해주면 된다.

## 개발환경 공유
```bash
pip freeze > requirements.txt
```
해당 명령어를 사용하면 가상환경에 설치된 패키지들의 정보를 빼내어 txt파일로 저장할 수 있다.
따로 저장하는 이유는 동일한 환경에서 서비스를 돌리기 위함이다. 만약 개발자가 의도한 환경에 맞추어진 서비스가 다른 패키지에 의해 정상적인 작동이 안될 수도있다 그런 상황을 방지하기 위함이기도하고, 작동함에 필요한 패키지를 인지시키기 위해서 이기도하다.

## 보안?
맨 위에서 `.gitignore`에 `my_settings.py`를 작성해야한다는 것을 이야기했다.
그럼 어떻게 작성하는 것일까?

```python
DATABASES = {
    'default' : {
        'ENGINE': 'django.db.backends.mysql',
        'NAME': 'DATABASE 명',
        'USER': 'DB접속 계정명',
        'PASSWORD': 'DB접속용 비밀번호',
        'HOST': '실제 DB 주소',
        'PORT': '포트번호',
    }
}

SECRET = {
        'secret':'시크릿키',
}
```
`settings.py`에 작성되어있던 내용 중 databases와 secret이 있었다. 하지만 `setttings.py`는 git을 업로드가 되어야한다.
그렇게 되면 두 내용에는 비밀번호와 암호화시 사용하는 키를 담고 있기때문에 보안에 취약해지게된다. 그러므로  `my_settings.py`에 해당 내용을 작성해야하는 것이다.

그럼 `settings.py`에서 빠진 내용들은 어떻게 참조하게될까?

```python
import my_settings

DATABASE = my_settings.DATABASES

SECRET = my_settings.SECRET['secret']
```
위와 같이 import하여 사용하는 방식을 이용한다.
