# Django 1

sudo apt install tree

tree 목록 보기 쉽게 만들어주는 플러그인이다.

asgi = 비동기식
wsgi = 웹서버, 만들때 반드시 필요한 모듈 장고,플라스크 등 다씀
urls = url만들어서 라우팅을 해준다


1. 프로젝프 만들 디렉토리 만들기
Django 프로젝트를 만들 디렉토리를 만든후
```python
django-admin startproject <프로젝트이름>
django-admin startproject <프로젝트이름> .
```
django-admin startproject <프로젝트이름> 을 하면 아래의 트리처럼 되면서 프로젝트 이름의 폴더를 
새로 만들고 그안에 파일을 생성하지만 끝에 .(dot)을 찍으면 현재 폴더에서 만들겠다는 뜻이 된다.

```bash
├── manage.py
└── test01
    ├── __init__.py
    ├── asgi.py
    ├── settings.py
    ├── urls.py
    └── wsgi.py
```
위와 같은 트리형태로 초기 설정이된다. 기본적으로 이 프로젝트에서 최상위 위치는 `manage.py`가 있는 위치가 된다.

초기 세팅이 되고나면, settings.py를 설정해 주어야한다.

`settings.py` = 프로젝트의 셋팅값에 따라 프로젝트의 방향 정해짐 (제일 먼저 보게됨)

    import os - manage.py가 있는 장소를 최상위 경로로 만들기위해 사용 (BASE_DIR)
    
    SECRET_KEY = 사용안함 보안위험 함부러 관리하지 말것

    DEBUG = True 오류페이지 보여줄지 여부, 개발할때만 사용, 배포할때 무조건 끌것

    ALOOWED_HOSTS = ['*'] - '*'(모두 들어올수 있게끔함), 한국의 ip대역 넣는 식으로 들어올 수 있는 사용자 제어함.

    INSTALLED_APPS = [] - django에 내장된 기능들로 admin, auth 사용하지 않는다.

    MIDDLEWARE = [] Auth,Csrf(위장 피하기위해서 사용, 위장페이지 검사) # 주석처리할것

    ROOT_URLCONF = 최상위 url 설정 'test01.urls' ex)localhost:8000/test01.url 호출됨

    TEMPLATES = HTML CSS 만들어주는 것이지만 지우지는 않는다.

    WSGI_APPLICANTION = 

    DATABASES = SQLITE3 관계형 데이터베이스, 파이썬 설치시 같이 설치되며 django에서 기본으로 지원

    AUTH_PASSWARD

    LANGUAGE_CODE = 지원언어

    TIME_ZONE = 'asia/seoul' utc 지원하지만 +9 -9 단순히 붙이는 걸로는 안됨

    USE_I18N = 다국어지원

    USE_L10N = 다국어지원

    USET_TZ = 서버 시간?

    STATIC_URL = 정적 페이지주소 파일 같은거 받을 때 사용
    
    'django.contrib.admin' 지우면 url 설정 바꿔줘야함 import admin으로 된것

세팅이 끝났으면

```django
python manage.py startapp user
```
startapp 명령어를 통해 만들기 원하는 앱을 만들 준비를 한다.

```bash
├── db.sqlite3
├── manage.py
├── test01
│   ├── __init__.py
│   ├── __pycache__
│   │   ├── __init__.cpython-38.pyc
│   │   ├── settings.cpython-38.pyc
│   │   ├── urls.cpython-38.pyc
│   │   └── wsgi.cpython-38.pyc
│   ├── asgi.py
│   ├── settings.py
│   ├── urls.py
│   └── wsgi.py
└── user
    ├── __init__.py
    ├── admin.py
    ├── apps.py
    ├── migrations
    │   └── __init__.py
    ├── models.py
    ├── tests.py
    └── views.py

```
그러면 이러한 형태가 되는데 test01의 urls.py를 user에 복사한다.

새로운 앱을 만들었는데 새로만들어진 앱은 django가 인식하지 못한다. setting.py에서 INSTALLED_APPS 에 추가해준다.
추가할 앱이름의 폴더가 user라면 'user'라고 작성하면 된다.

여기까지 설정을 마치면 데이터 베이스의 모델을 만들어 주어야한다. 그렇게 하기위해 `models.py`를 설정해 준다.

```python
# models.py
from django.db import models #orm 수행

class User(models.Model):  #User는 models.Model의 서브클래스임
    name     = models.CharField(max_length = 50)
    email    = models.CharField(max_length = 200)
    password = models.CharField(max_length = 400)
    created_at = models.DateTimeField(auto_now_add = True)
    updated_at = models.DateTimeField(auto_now = True)     #django의 >편의성기능 중 하나 now_add 최초 한번만 기록 수정안함, auto_now 업데이>트 될 때마다 업데이트함

    class Meta:
        db_table = 'users'  # 데이터 베이스 이름 지정, 기록안하면 "app이름_클래스명"으로 만듦
```
위는 간단한 데이터베이스 모델을 만든 것이다.

```django
python manage.py makemigrations <앱이름> 
```
models migration 할시 앱이름 꼭 작성해야한다. 그렇지 않으면 모든 앱들을 migration을 시켜 순서가 꼬여 곤란해 질 수 있다.

migration한 파일들은 해당앱 migrations폴더에 저장된다. makemigrations를 한다는 것은 해당 app의 models.py안 클래스에 코딩해놓은 테이블 기반으로 Django에서 쓸수있게 장고에서 따로 메타데이터로 만들어놓는 것이다. 그렇게해야 테이블구조 생김 migrations 폴더에 파일을생성한다.

```bash
python manage.py migrate user 0001
```
방금전 migration한 내용을 바탕으로 데이터베이스에 실제로 테이블을 만들어주는 것이 migrate이다.
