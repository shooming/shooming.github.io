# mongoDB의 Database 내용 조회하기
database에 내용을 작성했다면 그 내용을 확인해야 할 수 있어야한다.
mongoDB에서는 `find()`문을 사용하여 내용을 손쉽게 조회할 수 있다.

## Document 조회하기 ( db.COLLECTION_NAME.find(query. projection) )

`find()`로 검색한 내용들은 기준에 해당하는 Document들을 선택해서 cursor로 반환한다.
cursor 객체를 통해서 보이는 데이터를 `제한(limit)`하거나 `정렬(sort)` 할 수 있으며 10분동안 사용되지 않으면 만료된다.

```python
파이썬으로 검색해서 나온 cursor 객체
```
위는 pymongo를 사용하여 파이썬에서 mongdb에 find문을 보내 해당하는 내용들을 가져온 모습이다. 보면 cursor 객체가 반환되었음을 확인 할 수 있다.

|  parameter |   Type   |                                                                                    설명                                                                                    |
|:----------:|:--------:|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
|    query   | document | Optional(선택적).  다큐먼트를 조회할 때 기준을 정합니다. 기준이 없이 컬렉션에 있는 모든 다큐먼트를 조회 할때는 이 매개변수를 비우거나 비어있는 다큐먼트 { } 를 전달하세요. |
| projection | document | Optional. 다큐먼트를 조회할 때 보여질 field를 정합니다.                                                                                                                    |

### db.collection_name.find()
위 명령어는 해당 collection의 document를 조회하는 명령어이다.
```bash
db.나와라요.find() # db.나와라요.find({}) {}안에는 sql query를 넣지만 안넣으면 find()와 같다.
암튼 내용
```

### db.collecion_name.find().pretty()
위 명령어는 find를 통해 나온 검색결과를 보기 좋은 형태로 출력해준다.
만약 python에서 pymongo를 사용하고 있다면 `pprint`를 사용하여 비슷한 출력결과를 얻을 수 있다.

```bash
db.나와라요.find().pretty() # pprint.pprint(질의 결과)
암튼 내용 
```

### db.collection_name.({"name" : "김수환무"}).pretty
```bash
db.나와라요.find({"name" : "김수환무"}).pretty
대충 내용
```

### find문의 논리 연산자
mongodb는 비교 연산자와 논리 연산자를 통해서 원하는 내용들을 검색해올 수 있다.

#### 비교 연산자
| operator |                          설명                          |
|----------|------------------------------------------------------|
| $eq      | (equals) 주어진 값과 일치하는 값                       |
| $gt      | (greater than) 주어진 값보다 큰 값                     |
| $gte     | (greather than or equals) 주어진 값보다 크거나 같은 값 |
| $lt      | (less than) 주어진 값보다 작은 값                      |
| $lte     | (less than or equals) 주어진 값보다 작거나 같은 값     |
| $ne      | (not equal) 주어진 값과 일치하지 않는 값               |
| $in      | 주어진 배열 안에 속하는 값                             |
| $nin     | 주어빈 배열 안에 속하지 않는 값                        |

#### 논리 연산자
| operator |                  설명                  |
|----------|--------------------------------------|
| $or      | 주어진 조건중 하나라도 true 일 때 true |
| $and     | 주어진 모든 조건이 true 일 때 true     |
| $not     | 주어진 조건이 false 일 때 true         |
| $nor     | 주어진 모든 조건이 false 일때 true     |

### db.collection_name.find({"age" : {\$gt: 10, $lt:25}}).pretty()
위 내용은 age가 10초과 25미만에 해당하는 값들을 찾아오는 find문이다.
```bash
db.나와라요.find({"age" : {$gt: 10, $lt:25}}).pretty()
대충 내요이 나올 예정임
```

### db.collection_name.find({"name":{$in:["김수한무", "고길동", "기영이"]}})
위 내용은 `$in`배열에 포함된 Document를 조회하는 find문이다.
```bash
db.나와라요.find({"name" : {$in : ["김수한무", "고길동", "기영이"]}})
대충 해당하는 내용
```
### db.collecion_name.find({$and:[{"name" : "기영이"}, {"age" : {gt : 10}}]})
위 내용은 `$and`문으로 이름이 기영이 이면서 나이가 10을 초과하는 document를 조회하는 find문이다.

```bash
db.나와라요.find({$and:[{"name" : "기영이"}, {"age" : {$gt : 10}}]})
db.나와라요.find({"name":"기영이"}, {"age":{$gt : 10}}) # 위와 같은 내용이지만 이렇게도 사용할 수 있다.
```

### db.collection_name.find({$or : [{"name" : "고길동"}, {"age" : "20"}]})
위 내용은 `$or`문으로 이름이 고길동 이거나 나이가 20인 document를 찾아오는 find문이다.
```bash
db.나와라요.find({$or : [{"name" : "고길동"}, {"age" : "20"}]})
```



### 정규표현식을 사용한 find
`$regex` 연산자를 사용하면 정규표현식을 동하여 Document의 내용을 찾아오는 것이 가능하다.
정규식만 사용한다면 따로 `$regex`를 사용하지 않아도 된다.
```bash
{ <field>: { $regex: /pattern/, $options: '<options>' } }
{ <field>: { $regex: 'pattern', $options: '<options>' } }
{ <field>: { $regex: /pattern/<options> } }
{ <field>: /pattern/<options> }
```
그리고 사용할수 있는 옵션은 몇 가지가 있다. 옵션의 내용은 아래와 같다.

| option |                             설명                            |
|:------:|-----------------------------------------------------------|
|    i   | 대소문자 무시                                               |
|    m   | 정규식에서 anchor(^) 를 사용 할 때 값에 \n 이 있다면 무력화 |
|    x   | 정규식 안에있는 whitespace를 모두 무시                      |
|    s   | dot (.) 사용 할 떄 \n 을 포함해서 매치                      |

#### db.collecion_name.find({"name" : /^김/})
위 내용은 name에 성이 김으로 시작하는 사람들만 find해서 가지고오는 find문이다.
```bash
db.나와라요.find({"name" : /^김/})
```

### where 연산자를 사용한 find
`$where`연산자를 사용하면 javascript expression(표현식)을 사용할 수 있다.
```bash
db.나와라요.find({ $where : "this.name.length === 4" }) # 이름의 길이가 4글자인 document 검색
```

### 내부 객체 find
내부에 객체를 검색하는 방법에는 몇 가지가 존재한다.
1. `$elemMatch`연산자로 Embedded Document배열을 하용할 때 사용한다. Embedded Documents는 document의 key에 대한 값이 document가 되는 것이다.

```bash
db.나와라요.find( { "school": { $elemMatch: { "name": "알파 초등학교" } } } ) # School중에 name의 내용이 알파 초등학교인 document를 조회한다.
```

2. Embedded Document 배열이 아니라 Document의 "name" 처럼 한개의 Embedded Document일 경우
```bash
{
    "username" : "show",
    "name" : {"first" : "megi" "last" : "oh"},
    "language" : ["korean", "english", "chinese"]

}
```
위의 document와 같이 Embedded Document가 배열이 아니라 한개 인경우와 document가 아니라 arry 형태가 두가지가 있다.

한개의 document 조회할때
```bash
db.users.find({ "name.first": "megi"}) # name의 값의 docment의 first가 megi인 것을 find
```

document의 arry 조회할때
```bash
db.users.find({ "language": "korean"}) # 키가 language이면서 korean을 값으로 가지는 것 find
```

### projection
해당 projection은 find() 메소드의 parameter로 qurey의 결과값에서 보여질 field를 결정한다.

#### db.collecion_name.find({}, {"_id" : true, "name" : true})
위 내용은 전체 document를 가져오며 _id와 name만 가지고 오겠다는 내용이다.
```bash
db.나와라요.find({}, {"_id" : true, "name" : true})
db.나와라요.find({}, {"_id" : 1, "name" : 1}) # 1만 적게되면 true와 같이 작동한다.
```

#### $slice 연산자
Embedded Document 내용을 읽을때 limit을 설정하기위 한 연산자이다.

```bash
db.나와라요.find( {"name" : "김수한무"}, {"school" :  {$slice : 1}}) # 김수한무의 school은 한개만 나오게 한다.
```

#### $elemMatch 연산자
Embedded Document이 존재하는 경우 해당하는 document의 원하는 내용들만 출력하게 해줄 수 있다.
```bash
db.나와라요.find( { "school": { $elemMatch: { "name": "알파 초등학교" } } },
                 { "name" : true,
                   "school" : { "name" : "알파 초등학교" },
                   "school.name" : true,
                   "school.location" : true
                 }
)
```

### findOne 조회
해당 명령어는 단 find로 조회한 내용 중 하나만 찾고 싶을때 사용한다. find()[0]으로 가장 첫번째것을 출력해 준다.

```bash
db.나와라요.findOne({ "age" : 10}) # age가 10인 내용들 중 가능 첫번째 값을 출력함.
```
