# Array와 List의 차이

## Array? List? 같은거 아닌감?
Array는 data + index 의 형태로 되어 있고 List도 형태는 같다.
하지만 Array는 index의 주소가 중요하지만 List는 data의 순서가 더 중요하게 작용한다.

### Array

특징
- ⭐️Array는 크기를 지정해 주어야한다.
- Array는 내부적으로 연속된 형태의 배열 구조를 가지고 있다.
- ⭐️Array는 내부의 각각의 elements에 index를 부여하고 있다.
- Array는 조회속도가 빠르다.
- Array는 `cache hit`의 가능성이 커져서 성능 향상에 도움이된다.

단점
- Array는 정적이기 때문에 컴파일 이전에 크기를 정해주어야 한다.
- ⭐️Array는 크기가 지정되어 있기 때문에 내부 element중 하나를 지우면 빈공간을 남겨 메모리 낭비가 생길 수 있다.
- 컴파일 이후 배열의 크기를 바꿀 수 없다.

한계
- 배열의 길이를 바꿀 수 없기 때문에 길이가 변경 가능한 배열의 경우 아래의 단계를 따른다.
  1. 기존의 배열 보존
  2. 새로운 길이 지정된 배열 할당 (메모리 있는지 탐색 필요함)
  3. 데이터의 복사 진행
  4. 기존 배열 삭제
  5. 총 3번의 작업 + 메모리 탐색이 필요하므로 리소스 낭비가 큼

- 한계 극복을 위해 linked list 자료형을 활용함.
- 조건문 사용하면 빈 element를 제외가능 하지만 조건문 많으면 좋지 않음

### List

특징
- List는 순서가 있는 데이터의 집합이다.
- List는 불연속적으로 메모리 공간을 차지한다.
- ⭐️List는 인덱스가 없어 포인터를 통해 다음 element에 접근하게 되며 동적으로 표현된다.
- ⭐️데이터의 삭제 삽입이 용이하다.
- ⭐️동적이기 때문에 크기가 정해져 있지 않음
- 메모리의 재상용성이 좋고 관리가 편하다.

단점
- ⭐️순차성이 보장이 어렵기 때문에 검색성능 떨어지게된다.
- 포인터가 다음 데이터를 가르키기 때문에 메추가적인 메모리 공간이 발생한다.

## 각 언어별 List 특징

### C
- 리스트를 지원하지 않고 배열을 지원한다.
- 리스트를 사용하려면 직접 만들거나 라이브러리 사용해야함

### Java Script
- C 패밀리 랭귀지(C언어의 파생어 제품)
- 자바스크립트에서는 배열에 리스트의 기능이 포함되어 있다.

### Python
- 기본적으로 리스트 제공, 배열은 제공하지 않는다.
- 파이썬에서는 리스트가 배열임
- 파이썬의 리스트는 크기가 가변적, 어떤 원소 타입이던 저장이 가능한 장점이있지만 C의 Array보다 메모리를 더 많이 필요로 한다.

### Java
- 배열과 리스트를 모두 지원, 두 가지 완전분리 되어 있음
- 개발자가 원하는 구조 선택가능
- 자바는 2가지 형태의 리스트 지원
  - ⭐️LinkedList / ArrayList
  - 똑같은 기능(메소드)를 가진 리스트가 2가지 존재
  - 각각의 장단점이 분명하다.
